# ADR 1: Record Architecture Decisions

## Status

Accepted

## Context

We need a standard, collaborative way to track and document major engineering decisions made during the design of KorriPay.

## Decision

We will use Architecture Decision Records (ADRs) following the standard template. All ADRs will be stored in markdown format inside `docs/adr/` directory.

## Consequences

- Major changes to design patterns, libraries, or smart contract architectures must have a corresponding ADR.
- ADRs are committed to git to ensure history transparency.
